VirConv Framework（整体框架）¶

VirConv Framework 用于 统一组织并调度各阶段算子与模块，
构成从点云输入到融合体素特征输出的完整前向计算流程。

该框架本身不引入新的计算逻辑，而是按照既定顺序串联各阶段算子。

框架组成¶

VirConv Framework 主要由以下部分构成：

  1. Virtual Point Generator
    用于生成伪点云（Demo 中可使用 Simple 实现）。

  2. Voxel Preparation Stage
    对点云与伪点云进行融合与体素化处理。

  3. VirConv Stage
    执行 VirConv Block 的核心计算。

前向执行流程¶

框架在前向过程中依次完成以下步骤：

  1. 生成或接收伪点云
  2. 调用 Voxel Preparation Stage 构建体素级输入
  3. 调用 VirConv Stage 进行特征融合计算
  4. 输出融合后的体素特征

class VirConvFramework(torch.nn.Module):
    """
    VirConv Framework。

    该框架用于将 VirConv 方法中的各个算子与网络模块按照既定顺序
    进行组织与调度，构成从 LiDAR 点云与伪点云输入到融合体素特征输出的
    完整前向计算流程。

    本框架本身不引入新的几何或融合逻辑，而是作为一个 **Pipeline / Orchestrator**，
    串联以下两个主要阶段：

    - Voxel Preparation Stage（体素准备阶段）
    - VirConv Block Stage（VirConv 核心计算阶段）

    所有不可学习算子均通过 `torch.ops.virconv.*` 形式调用，
    可学习模块则以 `nn.Module` 的形式注册在框架中。

    ---
    Args:
        virtual_point_generator (VirtualPointGeneratorInterface):
            伪点云生成器接口，用于从图像或其他模态信息中生成虚拟点云。
            在 Demo 中可使用 Simple 实现。

        voxel_size (Tuple[float, float, float]):
            体素大小 (vx, vy, vz)，用于点云体素化。

        point_cloud_range (Tuple[float, float, float, float, float, float]):
            点云空间范围 (x_min, y_min, z_min, x_max, y_max, z_max)。

        bin_edges (torch.Tensor):
            距离分箱的边界，用于 distance_binning 算子。

        input_discard_rate (float):
            STVD 输入阶段的点云丢弃比例，用于抑制冗余虚拟点。

        layer_discard_rate (float):
            STVD Layer 内部的体素丢弃比例。

        feat_dim (int):
            体素特征维度。
    """

    def __init__(self,
                 virtual_point_generator: VirtualPointGeneratorInterface,
                 voxel_size: Tuple[float, float, float],
                 point_cloud_range: Tuple[float, float, float, float, float, float],
                 bin_edges: torch.Tensor,
                 input_discard_rate: float,
                 layer_discard_rate: float,
                 feat_dim: int):
        super().__init__()

        self.virtual_point_generator = virtual_point_generator
        self.voxel_size = torch.tensor(voxel_size, dtype=torch.float32)
        self.point_cloud_range = torch.tensor(point_cloud_range, dtype=torch.float32)
        self.bin_edges = bin_edges
        self.input_discard_rate = input_discard_rate
        self.layer_discard_rate = layer_discard_rate

        # ===== 可学习模块 =====
        self.nrconv3d = NRConv3D(feat_dim, feat_dim)
        self.nrconv2d = NRConv2D(feat_dim, feat_dim)
        self.fusion = FeatureFusion(feat_dim, feat_dim, feat_dim)
        self.spconv3d = SparseConv3D(feat_dim, feat_dim)

    def forward(self,
                lidar_points: torch.Tensor,
                image: Optional[torch.Tensor] = None,
                calib: Optional[dict] = None,
                proj_matrix: Optional[torch.Tensor] = None,
                training: bool = True):
        """
        执行完整的 VirConv 前向流程。

        该函数按照 VirConv 的标准计算顺序，依次完成：
        1. 伪点云生成
        2. 点云与伪点云融合
        3. 体素化与 STVD 输入构建
        4. VirConv Block 核心计算
        5. 输出融合后的体素特征

        ---
        Args:
            lidar_points (torch.Tensor):
                输入 LiDAR 点云，shape 为 (N, C)，
                由数据集或 3D 检测器预处理模块提供。

            image (torch.Tensor, optional):
                输入图像数据，用于伪点云生成或 2D 投影相关计算。
                在 Fake Demo 中可为 None。

            calib (dict, optional):
                标定参数字典，通常包含 LiDAR 与 Camera 之间的外参及投影矩阵。

            proj_matrix (torch.Tensor, optional):
                3D → 2D 投影矩阵，用于 projection 算子。

            training (bool):
                是否处于训练阶段，用于控制 STVD Layer 中的随机丢弃行为。

        ---
        Returns:
            out_feat (torch.Tensor):
                融合后的稀疏体素特征，作为 VirConv Block 的最终输出，
                可直接对接下游 3D 检测头或其他任务模块。
        """
        # ===== 0. Virtual points (interface) =====
        virtual_points = self.generate_virtual_points(
            lidar_points, image=image, calib=calib
        )

        # ===== Stage A: Voxel Preparation (torch.ops) =====
        points = torch.ops.virconv.point_fusion(          # 1
            lidar_points, virtual_points
        )

        points = torch.ops.virconv.point_align(           # 2
            points
        )

        voxel_coords, voxel_features = torch.ops.virconv.voxelization(  # 3
            points,
            self.voxel_size,
            self.point_cloud_range
        )

        bin_ids = torch.ops.virconv.distance_binning(     # 4
            voxel_coords,
            self.voxel_size,
            self.point_cloud_range,
            self.bin_edges
        )

        voxel_coords, voxel_features = torch.ops.virconv.stvd_input(  # 5
            voxel_coords,
            voxel_features,
            bin_ids,
            self.input_discard_rate
        )

        # ===== Stage B/C: VirConv Block =====
        voxel_coords, voxel_features = torch.ops.virconv.stvd_layer(  # 6
            voxel_coords,
            voxel_features,
            self.layer_discard_rate,
            training
        )

        feat_3d = self.nrconv3d(voxel_features)            # 7

        pixel_coords = torch.ops.virconv.projection(       # 8
            voxel_coords,
            self.voxel_size,
            self.point_cloud_range,
            proj_matrix
        )

        feat_2d = self.nrconv2d(feat_3d, pixel_coords)     # 9
        fused_feat = self.fusion(feat_3d, feat_2d)         # 10
        out_feat = self.spconv3d(fused_feat)               # 11

        return out_feat
